# LandDecoded

Turns scanned/typed Bangla+English land-record PDFs (up to ~100 pages) into
structured, queryable fields — routing each page to the cheapest reliable
path, and reconciling two LLM providers by confidence instead of trusting
either one blindly.

## Why these documents are hard

- Mixed Bangla + English on the same page, sometimes the same line.
- Some pages carry a real Unicode text layer; others are scans/photos with
  skew and uneven lighting.
- Some carry legacy non-Unicode Bangla fonts (SutonnyMJ/Bijoy) whose bytes
  *look* like extractable text but decode to gibberish.

## Key design decisions

| Decision | Why |
|---|---|
| **Native text first, OCR only when needed** | The biggest cost lever is skipping the vision model entirely when possible. Each page's native text is validated with a character-ratio heuristic (Bangla block + Latin/digits vs. everything else) — built to catch legacy-font mojibake, not to require Bangla content. Only unreadable or image-bearing pages get preprocessed and OCR'd. On a 100-page document: the difference between ~2 LLM calls and ~200. |
| **No binarization in preprocessing** | Perspective correction → deskew → CLAHE contrast enhancement, but the image stays grayscale. Classic OCR engines (Tesseract) want binarized input; a multimodal LLM does better with the grayscale detail binarization would destroy. |
| **Dual-provider ensemble, not a bigger single model** | `FillSchema` runs concurrently against Claude and Gemini. Per field: higher self-reported confidence wins outright; on a tie, agreement keeps the value, disagreement nulls it — treated as a real signal, not resolved by arbitrary preference. |
| **Confidence gating instead of a human-review queue** | An earlier design queued low-confidence fields for manual review. Removed in favor of something simpler: every field carries its own LLM-assigned `LOW`/`MED`/`HIGH` confidence, and `LOW` is nulled immediately — the pipeline never silently keeps a value the model itself flagged as likely hallucinated. |
| **List fields merge by intersection, not agree/null** | vendors/vendees are lists of people. A tie doesn't null the whole list on any mismatch — it keeps the names both providers independently found, so partial agreement isn't thrown away like a scalar mismatch would be. |
| **`run_pipeline` is async now, not later** | `asyncio.run()` can't be called from inside an already-running event loop — which any async web framework (FastAPI, Starlette) fronting this pipeline provides. Going async now means the planned API layer can `await run_pipeline(...)` directly instead of forcing a rewrite when it lands. |

## Pipeline flow

```
PDF ─▶ split_pdf_pages (PyMuPDF)
        │  native text extraction + 200 DPI render, per page
        ▼
   text valid & no embedded image?
        ├─ yes ─▶ use native text, zero LLM cost
        └─ no  ─▶ preprocess_image ─▶ batched ExtractOcrBatch (Claude)
        ▼
   merge native + OCR'd text, in page order, into one document blob
        ▼
   FillSchema ×2, concurrently (Claude + Gemini)
        ▼
   merge_dual_provider_fields (confidence-ranked, per-field reconciliation)
        ▼
   JSON result
```

## Repo structure

```
src/
  ocr_pipeline/
    ingestion.py      # PDF → pages: native text extraction + routing
    preprocessing.py  # image cleanup for OCR-bound pages
    review.py         # confidence gating + dual-provider reconciliation
    pipeline.py        # wires the above + the two BAML calls, async
  baml_src/           # BAML source: schema, prompts, client definitions
  baml_client/        # generated by `baml-cli generate` — never hand-edited
tests/
  test_*.py                    # unit tests, no real PDFs or LLM calls
  test_*_real_document.py      # sanity checks against a real sample PDF
                                # (gitignored, local-only — skipped on a
                                # fresh clone)
docs/superpowers/
  specs/   # design docs — the "why", written before the "how"
  plans/   # task-by-task implementation plans generated from each spec
```

Every module went through the same cycle: written design spec → reviewed
implementation plan → test-first implementation → multi-angle code review.
`docs/superpowers/` keeps that trail, including design paths that were
tried and reverted (e.g. bounding-box capture, the human-review queue).

## Schema

`FillSchema` returns one `LandRecordFields` object per document — each
field paired with its own `ConfidenceLevel` (`LOW`/`MED`/`HIGH`):

| Field | Notes |
|---|---|
| `district`, `block`, `mouza`, `police_station`, `registry_location` | Location details |
| `plot_no` | Whichever of LR/RS/CS plot number is present |
| `deed_no` | Deed number |
| `registration_year`, `_month`, `_day` | Separate fields — BAML has no native `datetime` type |
| `total_area`, `land_unit` | Area transferred + its unit (decimal/acre/bigha/katha) |
| `vendors`, `vendees` | Lists of people, whatever info is extractable per person |
| `deed_type` | e.g. gift, sale, partition |

## Setup & commands

Requires Python >= 3.13, [uv](https://docs.astral.sh/uv/), and
`ANTHROPIC_API_KEY`/`GOOGLE_API_KEY` in the environment (read automatically
by BAML's generated clients — no code change needed to configure them).

| Command | Does |
|---|---|
| `uv sync` | Install dependencies |
| `uv run pytest -v` | Run the full test suite |
| `uv run pytest tests/test_<name>.py -v` | Run one test file |
| `uv run baml-cli generate --from src/baml_src` | Regenerate the BAML client after editing `src/baml_src/*.baml` |

## Testing philosophy

- No test makes a real LLM call — every `ExtractOcrBatch`/`FillSchema` call
  site is mocked.
- A few tests additionally validate against a real scanned document, but
  that fixture is gitignored/local-only (it would otherwise contain real
  personal data) — those tests skip cleanly on a fresh clone.

## Status

Ingestion, preprocessing, review, and pipeline wiring are implemented and
tested. Not yet built: an API layer around `run_pipeline` (planned next, as
a microservice endpoint) and any persistence beyond the JSON string it
currently returns.
